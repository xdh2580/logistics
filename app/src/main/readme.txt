待解决的bug：
1.出库中数量问题，能超过库存数量
2.输入格式问题，新增货品功能以实现使用异常捕获，其他地方可参照，增强健壮性
3.模拟订单中生成的货品可能有重复
4.重要：导出数据库后，数据库会更新为最初，因为使用的是renameTo方法---已修复
5.当拣货订单中的货品被删除之后再进行拣货会出现异常
6.修改货品信息时输入的格式异常会crash
7.导入数据库时，目录下只能有一个db文件
8.导入导出数据库时缺少相应的操作步骤提示

心得：
1.StringBuild的append()方法后不要使用+拼接字符串，否则会出错，因为append方法就是为了解决String += String的效率问题的。
2.SQLite中的Cursor千万不能使用Cursor == null 来判断是否为空，即便Cursor中什么记录都没有，他也不会是空（已测试）。
判断是否为空的方法是 Cursor.getCount()这么一个简单的函数，如果是0，表示Cursor为空；如果非0，则表示Cursor不为空。
而且当查询到的cursor为空还继续对其进行操作如取数据的话，程序会直接crash，还没有Error报出来，比较难搞。因此多注意对查询结果进行判断，增强程序健壮性。
